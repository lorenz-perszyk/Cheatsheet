SEMANTICS IN HTML

HTML tags are seperated into semantic and non-semantic tags.
In order to keep the code accesible, easy to read and maintain, it is
considered best practice to use semantic tags whenever possible.

Non-Semantic
Tag Function
<div> The generic block container for flow content.
<span> A generic inline container for phrasing content
<embed> Embeds external content at the specified point in the document. This element is deprecated and should be avoided when possible.
Semantic
Tag Function
<header> The header represents introductory content, typically a group of introductory or navigational aids.
<main> This element represents the dominant content of the body of a document.
<footer> The footer typically contains information about the author of the section, copyright data or links to related documents.
<nav> Common examples of navigation sections are menus, tables of contents, and indexes.
<table> This element represents information presented in a two-dimensional table comprised of rows and columns of cells containing data.
<h1> to <h6> elements represent six levels of section headings. <h1> is the highest section level and <h6> is the lowest level.
<p> HTML paragraphs can be any structural grouping of related content, such as images or form fields.
